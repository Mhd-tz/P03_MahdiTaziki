Process Analysis

Documenting the iterative design decisions, technical constraints, and workflows behind the construction of this portfolio system.

01. Developing a Minimalist Identity & Responsive Interactions

Context & Objective

The core objective for P03 was to establish an identity that felt authentic, highly professional, and distinctly removed from standard web templates. Feedback indicated early drafts felt too generic. The new directive was to formulate a highly premium, editorial layout focusing on reduction rather than addition, complemented by intricate micro-interactions.

Design Strategy & Choices

I structured a high-contrast typographic hierarchy relying heavily on Playfair Display for structural authority and Inter for legibility. The primary aesthetic pivot removed standard geometric card borders and instead introduced a bespoke "Muted Purple" (`#695681`) as a brand accent. To elevate the interaction model, CSS keyframe animations (`@keyframes revealUp`) were introduced to stage elements sequentially as the user initially loads the site.

Challenges & Iterations

The primary hurdle was balancing a sparse, minimal aesthetic with necessary UI feedback. Without traditional button borders, I had to ensure hover states still felt engaging. The solution was implementing precise transform: scaleX(1) animations scaling out from the center on nav links, and dramatic transform: translateY(-8px) lifts combined with deep shadowing (var(--shadow-lg)) on the project cards to enforce interactivity. 

.project-card:hover {
  transform: translateY(-8px);
  box-shadow: var(--shadow-lg);
}

.nav-link:hover::after {
  transform: scaleX(1);
  transform-origin: bottom left;
}

02. Building the "About Me" Modal Architecture

Context & Objective

Initially, contact and bio information were dispersed throughout the main index layout inline. To refine the layout and provide a cleaner reading path, the logic was refactored to house the "About Me" logic within an immersive, full-screen modal triggerable globally from the top navigation.

Implementation Workflow

The technical structure required rendering a dual-pane modal (`.about-modal`) fixed in the window center, alongside a `backdrop-filter` enabled `.modal-overlay`. A custom vanilla JavaScript event listener ecosystem was implemented to control state mapping: clicking the trigger opens the GUI, disabling `body` overflow scrolling, while clicking the cross, hitting the `Escape` key, or clicking the opaque backdrop correctly closes the menu mapping and restores scrolling.

Challenges & Iterations

Z-index management and event propagation (e.stopPropagation()) were critical hurdles. Clicking inside the modal initially triggered the overlay's close listener, instantly dismissing the modal unexpectedly. By carefully structuring the JS to ignore clicks propagating *through* the modal pane while strictly listening to the overlay's independent layer, standard expected modal behaviour was achieved. 

// Prevent click inside modal from closing the overlay
modalWindow.addEventListener('click', (e) => {
    e.stopPropagation();
});

// Close on Overlay Click
modalOverlay.addEventListener('click', () => {
    closeModal();
});
